# 09 — Privacy and Consent Playbook

Cousin Radio is built around family memories. That means privacy and consent are part of the product, not an afterthought.

## Product Principle

```text
Private by default. Public only by clear choice.
```

## Visibility States

Every song, album, and family space should have a visible privacy state.

```text
Private Draft: only creator can see it
Family Only: approved family members can see it
Invite Link: anyone with the link can listen
Public: discoverable on Cousin Radio
Archived: hidden from normal viewing
Deleted: removed according to policy
```

## Default Settings

Recommended defaults:

```text
New song: Private Draft
New album: Family Only
New family space: Private
New child profile: Private
Public publishing: Off
```

## Consent Moments

The app should show consent language at these moments:

1. Recording a voice note
2. Adding names or photos
3. Creating a song about another person
4. Sharing by invite link
5. Publishing publicly
6. Adding children or young family members
7. Deleting or unpublishing content

## Creation Consent Copy

Suggested copy:

```text
Use stories, names, and photos you are comfortable saving in your family space.
Your song starts private. You choose who can hear it.
```

## Public Publish Copy

Suggested copy:

```text
Public songs may be heard by anyone on Cousin Radio. Only publish if the people involved are comfortable with this memory being shared publicly.
```

## Invite Link Copy

Suggested copy:

```text
Anyone with this link may be able to listen. You can turn off the link later.
```

## Child Profile Copy

Suggested copy:

```text
Keep children’s songs private or family-only unless a parent or guardian intentionally chooses otherwise.
```

## Required Controls

### For Creators

- Create draft
- Edit song details
- Change visibility
- Generate invite link
- Disable invite link
- Request public review
- Unpublish public song
- Delete draft

### For Family Owners

- Manage members
- Remove members
- Approve public publishing
- Change album visibility
- Archive songs
- Respond to removal requests

### For Recipients or Mentioned People

- Request removal
- Report concern
- Ask for public song to be made private

## Public Publishing Flow

Recommended flow:

```text
User chooses Publish Publicly
↓
App explains public visibility
↓
User confirms rights and comfort
↓
Moderation check runs
↓
Human review if needed
↓
Song becomes public only after approval
```

## Data Minimization

Do not collect extra personal details that the product does not need.

Avoid requiring:

- full legal names
- exact birthdays
- addresses
- school names
- private phone numbers
- sensitive identifiers

Use display names and relationship labels instead.

## Deletion and Unpublish

Deletion should be clear and respectful.

Recommended actions:

```text
Unpublish: remove from public discovery but keep in family space
Archive: hide from normal family view
Delete: remove according to retention policy
Disable link: make an invite link stop working
```

## Trust Checklist

Before beta:

- Every song has a visible visibility label.
- Invite links can be disabled.
- Public publishing requires confirmation.
- Public songs can be unpublished.
- Drafts can be deleted.
- Family owner permissions are clear.
- No private song appears in public radio.

## Long-Term Principle

Cousin Radio should feel like a safe family home, not a content platform trying to expose private life.
